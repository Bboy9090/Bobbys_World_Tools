Make run_interactive() resilient

Right now, youre using subprocess.run(..., check=True) for interactive tools. Thats fine, but it can turn normal failures into stack traces unless your try/except fully wraps it.

This version is sturdier:

py
import shlex
import subprocess
from typing import List

def run_interactive(cmd: List[str]) -> bool:
    log(f"[EXEC] Running: {' '.join(shlex.quote(x) for x in cmd)}")
    try:
        completed = subprocess.run(cmd, check=False)
        if completed.returncode != 0:
            log(f"[ERROR] Tool exited with code: {completed.returncode}")
            return False
        return True
    except FileNotFoundError:
        log(f"[ERROR] Binary not found: {cmd[0]}")
        return False
    except Exception as e:
        log(f"[ERROR] Execution failed: {e}")
        return False

1) useWs.ts + use-flash-progress-websocket.ts

  • Naming the function gives you a stable self for retries (connectSocket()), so reconnect logic doesnt break or call the wrong closure.

  • You also added the missing guard in one place: dont reconnect if OPEN or CONNECTING. That prevents duplicate sockets and storm reconnect loops.

Keep this pattern.

One more improvement I recommend: make sure your timeout callback always calls the latest connect function (not a stale render). The cleanest way is a ref:

ts
const connectRef = useRef<() => void>(() => {});
connectRef.current = connectSocket;

reconnectTimeoutRef.current = window.setTimeout(() => {
  reconnectAttemptsRef.current += 1;
  connectRef.current();
}, reconnectInterval);

This avoids works most of the time bugs when state/props change.

2) use-mobile.ts

Change: useState<boolean | undefined>(undefined) useState<boolean>(getIsMobile) with SSR guard + matchMedia listener

Why its good:

  • You removed the undefined state (which causes UI flicker and weird conditional rendering).

One caution: older Safari used mql.addListener/removeListener. If you care about older iOS Safari, add a fallback:

if (mql.addEventListener) mql.addEventListener("change", onChange);
